v1.0.0 patch — Fix phantom injection + badge selectors (2026-05-16)
- Bug: panel injected in inbox list view (no thread open) — DOM fallback in getKey() was
  picking up data-legacy-thread-id from list row <tr> elements; fixed by gating fallback
  on div.adn.ads being present (reading pane only)
- Bug: fallback injection targets (div[data-legacy-thread-id], [role=main]) in
  findInsertionPoint() also matched list view; removed both — only div.adn.ads and
  pop-out [role=main] are valid injection points
- Bug: badge selectors too narrow; broadened findThreadRow() to try tr/non-tr variants
  of both data-thread-id and data-legacy-thread-id; broadened findBadgeTarget() with
  more td fallbacks
- Fix: badge approach entirely rewritten — Gmail tr.zA rows have no usable href links
  or data-thread-id attributes; badge matching now uses two strategies:
  (1) click interception + jslog decode: capture tr.zA on click, base64-decode the "1:"
      section of jslog to extract numeric thread ID (thread-f:NUMERICID from JSON payload),
      store in chrome.storage for persistence across refreshes
  (2) jslog scan: during polling, decode each visible tr.zA row's jslog and match against
      stored numeric IDs; badges td:nth-child(4) (subject column)
- Fix: previous extraction was broken — thread-f:NUMERICID is inside base64-encoded JSON
  in the jslog "1:" section, not in the raw attribute; regex against raw jslog never matched
